fully dressed use case:

1. [[Preface elements]]: Includes scope, goal level, and the primary actor.
2. [[Stakeholders and Interest List]]: Identifies who cares about the system and why.
3. [[Preconditions]]: What must always be true before the use case starts (assumed, not tested).
4. [[Postconditions]] (Success Guarantees): What must be true upon successful completion.
5. [[Main Success Scenario (Happy Path]]): The typical success path without any branching.
6. [[Extensions (Alternative Flows)]]: Branching scenarios for both success and failure, often labeled with letters (e.g., 3b).
7. [[Special Requirements]]: Related non-functional requirements or constraints.
8. [[Technology and Data Variations]]: Technical or data format constraints (e.g., using specific I/O devices).
